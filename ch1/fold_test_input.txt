<span id="line63"></span>
<span id="line64"></span></span><span>&lt;<span class="start-tag">p</span>&gt;</span><span>GCC allows the user to selectively enable or disable certain types of
<span id="line65"></span>diagnostics, and change the kind of the diagnostic.  For example, a
<span id="line66"></span>project<span class="entity"><span>&amp;</span>rsquo;</span>s policy might require that all sources compile with
<span id="line67"></span></span><span>&lt;<span class="start-tag">samp</span> <span class="attribute-name">class</span>="<a class="attribute-value">option</a>"&gt;</span><span>-Werror</span><span>&lt;/<span class="end-tag">samp</span>&gt;</span><span> but certain files might have exceptions allowing
<span id="line68"></span>specific types of warnings.  Or, a project might selectively enable
<span id="line69"></span>diagnostics and treat them as errors depending on which preprocessor
<span id="line70"></span>macros are defined.
<span id="line71"></span></span><span>&lt;/<span class="end-tag">p</span>&gt;</span><span>
<span id="line72"></span></span><span>&lt;<span class="start-tag">dl</span> <span class="attribute-name">class</span>="<a class="attribute-value">table</a>"&gt;</span><span>
<span id="line73"></span></span><span>&lt;<span class="start-tag">dt</span>&gt;</span><span></span><span>&lt;<span class="start-tag">a</span> <span class="attribute-name">id</span>="<a class="attribute-value">index-pragma_002c-diagnostic</a>"&gt;</span><span></span><span>&lt;/<span class="end-tag">a</span>&gt;</span><span></span><span>&lt;<span class="start-tag">span</span>&gt;</span><span></span><span>&lt;<span class="start-tag">code</span> <span class="attribute-name">class</span>="<a class="attribute-value">code</a>"&gt;</span><span>#pragma GCC diagnostic </span><span>&lt;<span class="start-tag">var</span> <span class="attribute-name">class</span>="<a class="attribute-value">var</a>"&gt;</span><span>kind</span><span>&lt;/<span class="end-tag">var</span>&gt;</span><span> </span><span>&lt;<span class="start-tag">var</span> <span class="attribute-name">class</span>="<a class="attribute-value">var</a>"&gt;</span><span>option</span><span>&lt;/<span class="end-tag">var</span>&gt;</span><span></span><span>&lt;/<span class="end-tag">code</span>&gt;</span><span></span><span>&lt;<span class="start-tag">a</span> <span class="attribute-name">class</span>="<a class="attribute-value">copiable-link</a>" <span class="attribute-name">href</span>="<a class="attribute-value" href="view-source:https://gcc.gnu.org/onlinedocs/gcc/Diagnostic-Pragmas.html#index-pragma_002c-diagnostic">#index-pragma_002c-diagnostic</a>"&gt;</span><span> <span class="entity"><span>&amp;</span>para;</span></span><span>&lt;/<span class="end-tag">a</span>&gt;</span><span></span><span>&lt;/<span class="end-tag">span</span>&gt;</span><span></span><span>&lt;/<span class="end-tag">dt</span>&gt;</span><span>
<span id="line74"></span></span><span>&lt;<span class="start-tag">dd</span>&gt;</span><span>
<span id="line75"></span></span><span>&lt;<span class="start-tag">p</span>&gt;</span><span>Modifies the disposition of a diagnostic.  Note that not all
<span id="line76"></span>diagnostics are modifiable; at the moment only warnings (normally
<span id="line77"></span>controlled by <span class="entity"><span>&amp;</span>lsquo;</span></span><span>&lt;<span class="start-tag">samp</span> <span class="attribute-name">class</span>="<a class="attribute-value">samp</a>"&gt;</span><span>-W<span class="entity"><span>&amp;</span>hellip;</span></span><span>&lt;/<span class="end-tag">samp</span>&gt;</span><span><span class="entity"><span>&amp;</span>rsquo;</span>) can be controlled, and not all of them.
<span id="line78"></span>Use </span><span>&lt;<span class="start-tag">samp</span> <span class="attribute-name">class</span>="<a class="attribute-value">option</a>"&gt;</span><span>-fdiagnostics-show-option</span><span>&lt;/<span class="end-tag">samp</span>&gt;</span><span> to determine which diagnostics
<span id="line79"></span>are controllable and which option controls them.
<span id="line80"></span></span><span>&lt;/<span class="end-tag">p</span>&gt;</span><span>
<span id="line81"></span></span><span>&lt;<span class="start-tag">p</span>&gt;</span><span></span><span>&lt;<span class="start-tag">var</span> <span class="attribute-name">class</span>="<a class="attribute-value">var</a>"&gt;</span><span>kind</span><span>&lt;/<span class="end-tag">var</span>&gt;</span><span> is <span class="entity"><span>&amp;</span>lsquo;</span></span><span>&lt;<span class="start-tag">samp</span> <span class="attribute-name">class</span>="<a class="attribute-value">samp</a>"&gt;</span><span>error</span><span>&lt;/<span class="end-tag">samp</span>&gt;</span><span><span class="entity"><span>&amp;</span>rsquo;</span> to treat this diagnostic as an error,
<span id="line82"></span><span class="entity"><span>&amp;</span>lsquo;</span></span><span>&lt;<span class="start-tag">samp</span> <span class="attribute-name">class</span>="<a class="attribute-value">samp</a>"&gt;</span><span>warning</span><span>&lt;/<span class="end-tag">samp</span>&gt;</span><span><span class="entity"><span>&amp;</span>rsquo;</span> to treat it like a warning (even if </span><span>&lt;<span class="start-tag">samp</span> <span class="attribute-name">class</span>="<a class="attribute-value">option</a>"&gt;</span><span>-Werror</span><span>&lt;/<span class="end-tag">samp</span>&gt;</span><span> is
<span id="line83"></span>in effect), or <span class="entity"><span>&amp;</span>lsquo;</span></span><span>&lt;<span class="start-tag">samp</span> <span class="attribute-name">class</span>="<a class="attribute-value">samp</a>"&gt;</span><span>ignored</span><span>&lt;/<span class="end-tag">samp</span>&gt;</span><span><span class="entity"><span>&amp;</span>rsquo;</span> if the diagnostic is to be ignored.
<span id="line84"></span></span><span>&lt;<span class="start-tag">var</span> <span class="attribute-name">class</span>="<a class="attribute-value">var</a>"&gt;</span><span>option</span><span>&lt;/<span class="end-tag">var</span>&gt;</span><span> is a double quoted string that matches the command-line
<span id="line85"></span>option.
<span id="line86"></span></span><span>&lt;/<span class="end-tag">p</span>&gt;</span><span>
<span id="line87"></span></span><span>&lt;<span class="start-tag">div</span> <span class="attribute-name">class</span>="<a class="attribute-value">example smallexample</a>"&gt;</span><span>
<span id="line88"></span></span><span>&lt;<span class="start-tag">pre</span> <span class="attribute-name">class</span>="<a class="attribute-value">example-preformatted</a>"&gt;</span><span>#pragma GCC diagnostic warning <span class="entity"><span>&amp;</span>quot;</span>-Wformat<span class="entity"><span>&amp;</span>quot;</span>
<span id="line89"></span>#pragma GCC diagnostic error <span class="entity"><span>&amp;</span>quot;</span>-Wformat<span class="entity"><span>&amp;</span>quot;</span>
<span id="line90"></span>#pragma GCC diagnostic ignored <span class="entity"><span>&amp;</span>quot;</span>-Wformat<span class="entity"><span>&amp;</span>quot;</span>
<span id="line91"></span></span><span>&lt;/<span class="end-tag">pre</span>&gt;</span><span></span><span>&lt;/<span class="end-tag">div</span>&gt;</span><span>
<span id="line92"></span>
<span id="line93"></span></span><span>&lt;<span class="start-tag">p</span>&gt;</span><span>Note that these pragmas override any command-line options.  GCC keeps
<span id="line94"></span>track of the location of each pragma, and issues diagnostics according
<span id="line95"></span>to the state as of that point in the source file.  Thus, pragmas occurring
<span id="line96"></span>after a line do not affect diagnostics caused by that line.
<span id="line97"></span></span><span>&lt;/<span class="end-tag">p</span>&gt;</span><span>
<span id="line98"></span></span><span>&lt;/<span class="end-tag">dd</span>&gt;</span><span>
<span id="line99"></span></span><span>&lt;<span class="start-tag">dt</span>&gt;</span><span></span><span>&lt;<span class="start-tag">code</span> <span class="attribute-name">class</span>="<a class="attribute-value">code</a>"&gt;</span><span>#pragma GCC diagnostic push</span><span>&lt;/<span class="end-tag">code</span>&gt;</span><span></span><span>&lt;/<span class="end-tag">dt</span>&gt;</span><span>
<span id="line100"></span></span><span>&lt;<span class="start-tag">dt</span>&gt;</span><span></span><span>&lt;<span class="start-tag">code</span> <span class="attribute-name">class</span>="<a class="attribute-value">code</a>"&gt;</span><span>#pragma GCC diagnostic pop</span><span>&lt;/<span class="end-tag">code</span>&gt;</span><span></span><span>&lt;/<span class="end-tag">dt</span>&gt;</span><span>
<span id="line101"></span></span><span>&lt;<span class="start-tag">dd</span>&gt;</span><span>
<span id="line102"></span></span><span>&lt;<span class="start-tag">p</span>&gt;</span><span>Causes GCC to remember the state of the diagnostics as of each
<span id="line103"></span></span><span>&lt;<span class="start-tag">code</span> <span class="attribute-name">class</span>="<a class="attribute-value">code</a>"&gt;</span><span>push</span><span>&lt;/<span class="end-tag">code</span>&gt;</span><span>, and restore to that point at each </span><span>&lt;<span class="start-tag">code</span> <span class="attribute-name">class</span>="<a class="attribute-value">code</a>"&gt;</span><span>pop</span><span>&lt;/<span class="end-tag">code</span>&gt;</span><span>.  If a
<span id="line104"></span></span><span>&lt;<span class="start-tag">code</span> <span class="attribute-name">class</span>="<a class="attribute-value">code</a>"&gt;</span><span>pop</span><span>&lt;/<span class="end-tag">code</span>&gt;</span><span> has no matching </span><span>&lt;<span class="start-tag">code</span> <span class="attribute-name">class</span>="<a class="attribute-value">code</a>"&gt;</span><span>push</span><span>&lt;/<span class="end-tag">code</span>&gt;</span><span>, the command-line options are
<span id="line105"></span>restored.
<span id="line106"></span></span><span>&lt;/<span class="end-tag">p</span>&gt;</span><span>
<span id="line107"></span></span><span>&lt;<span class="start-tag">div</span> <span class="attribute-name">class</span>="<a class="attribute-value">example smallexample</a>"&gt;</span><span>
<span id="line108"></span></span><span>&lt;<span class="start-tag">pre</span> <span class="attribute-name">class</span>="<a class="attribute-value">example-preformatted</a>"&gt;</span><span>#pragma GCC diagnostic error <span class="entity"><span>&amp;</span>quot;</span>-Wuninitialized<span class="entity"><span>&amp;</span>quot;</span>
<span id="line109"></span>  foo(a);                       /* error is given for this one */
<span id="line110"></span>#pragma GCC diagnostic push
<span id="line111"></span>#pragma GCC diagnostic ignored <span class="entity"><span>&amp;</span>quot;</span>-Wuninitialized<span class="entity"><span>&amp;</span>quot;</span>
<span id="line112"></span>  foo(b);                       /* no diagnostic for this one */
<span id="line113"></span>#pragma GCC diagnostic pop
<span id="line114"></span>  foo(c);                       /* error is given for this one */
<span id="line115"></span>#pragma GCC diagnostic pop
<span id="line116"></span>  foo(d);                       /* depends on command-line options */
<span id="line117"></span></span><span>&lt;/<span class="end-tag">pre</span>&gt;</span><span></span><span>&lt;/<span class="end-tag">div</span>&gt;</span><span>
